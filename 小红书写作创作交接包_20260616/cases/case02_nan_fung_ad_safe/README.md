# 案例 02：南丰新邨审核友好笔记

## 这个案例学什么

用 `humanizer-zh`、`dbs-xhs-title`、`dbs-content`、`dbs-hook` 这类文字 skill，把真实房源写成审核更稳的小红书笔记。

## 关键链路

```text
项目规则 -> 真实房源字段 -> 对标结构 -> 标题公式 -> 正文 -> 风险词回读 -> 发布前检查
```

## 重点

| 点 | 做法 |
|---|---|
| 房源事实 | 价格、面积、地铁、成交先留源 |
| 标题 | 不写神盘，不写闭眼买 |
| 正文 | 保留楼龄、估价、维修、按揭年期这类真实风险 |
| 审核 | 清掉收益承诺、身份/入学承诺、夸张喊单词 |

## 文件

| 文件 | 说明 |
|---|---|
| `draft.md` | 成稿样本 |
| `sample_research.md` | 同赛道结构拆解 |
| `mimic_brief.md` | 借鉴结构 |
| `pre_publish_check.md` | 发布前检查 |
