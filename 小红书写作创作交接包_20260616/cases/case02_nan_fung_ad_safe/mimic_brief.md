# 仿写 brief

## 用到的新 skill 规则

| skill | 本篇用法 |
|---|---|
| `dbs-xhs-title` | 标题覆盖数字锚定、身份代入、风险提醒 3 类触发器，控制在 20 字内 |
| `dbs-content` | 先判断选题是否成立：真实房源 + 成交锚点 + 明确风险 |
| `dbs-hook` | 首句用“能看，但别按叫价直接冲”做独立 hook |
| `humanizer-zh` | 去掉 AI 汇报腔、夸张词、公式化排比 |

## 对标拆解

| 维度 | 本篇执行 |
|---|---|
| 标题结构 | `600万` + `太古/港岛东` + `2房` + `成交` |
| 首句判断 | 能看，但别按叫价直接冲 |
| 图片分工 | 图 1 停留，图 2 适合谁，图 3 成交与风险 |
| 证据类型 | 太古站约 6 分钟、391 呎、同屋苑成交、48 年楼龄 |
| 风险写法 | 不吓人，直接列先查什么 |
| 承接动作 | 留言预算 + 用途 + 区域 |

## 标题候选

| 标题 | 触发器 | 张力点 |
|---|---|---|
| `600万太古2房先看成交` | 数字锚定 | 总价、地铁、成交 |
| `港岛东600万2房能不能看` | 场景条件 | 区域、预算、疑问 |
| `大陆买家看太古先别急冲` | 身份代入 | 人群、风险 |
| `600万买港岛东先查这3点` | 风险提醒 | 预算、行动 |
| `太古站6分钟这套别只看价` | 认知提醒 | 地铁、反差 |
| `港岛600万两房真实门槛` | 数字锚定 | 区域、预算、人群 |

最终选用：`600万太古2房先看成交`。

原因：信息密度高，不夸张；既有点击点，也先把用户带到成交和风险上。
