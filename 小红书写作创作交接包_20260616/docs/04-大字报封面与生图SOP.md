# 大字报封面与生图 SOP

## 成功方向

小红书房产封面要先让人停下来。当前最有效的一类是：

```text
真实房间全屏铺底 + 黄白超粗中文大字 + 厚黑描边 + 硬阴影 + 红白小贴纸事实
```

## 画面标准

| 元素 | 要求 |
|---|---|
| 画幅 | 3:4 / 1080x1440 |
| 底图 | 授权真实图或真实图约束下的生图底图 |
| 主标题 | 8-16 个字，像搜索流大字报 |
| 颜色 | 黄/白主字，黑色厚描边，红白贴纸补事实 |
| 信息量 | 只放 1 个主问题 + 2 个关键事实 |
| 禁止 | 底部黑条、黑色价格栏、软阴影高级感、HTML 信息卡感 |

## 生图模型该负责什么

| 交给生图 | 不交给生图 |
|---|---|
| 封面视觉冲击 | 编造房源事实 |
| 房间氛围和构图 | 生成假豪装、假海景 |
| 大字报文字效果 | 复杂中文长段落 |
| 多方向草图 | 真实地图、地铁线路、成交表 |

## 大字报提示词骨架

可直接看 `prompts/生图模型_大字报封面.md`。

核心写法：

```text
基于这张真实房间图，做一张小红书房产封面。
画幅 3:4，1080x1440。
保留真实小户型现场感，不要把房间改成豪装。
主标题做搜索流大字报：黄白超粗中文，厚黑描边，硬阴影。
主标题：[主标题]
小贴纸事实：[事实1]、[事实2]
不要底部黑条、不要黑色价格栏、不要软阴影高级感、不要歪斜贴片。
```

## 发布前检查

| 检查 | 通过标准 |
|---|---|
| 中文字 | 没错字、没乱码、没多字 |
| 房间状态 | 没被美化成不存在的装修 |
| 数字 | 价格、面积、距离和楼层都能追溯 |
| 观感 | 像小红书封面，不像公司 PPT |
| 风险 | 没写最低价、稳赚、必买、捡漏 |
