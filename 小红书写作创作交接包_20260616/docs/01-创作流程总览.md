# 创作流程总览

## 一句话版本

先拿真实资料，再学同赛道结构，然后写成小红书用户会看的内容，最后把来源、提示词、成稿和审核都留在一个包里。

## 标准顺序

| 步骤 | 做什么 | 产物 |
|---|---|---|
| 1 | 收集真实资料 | `source_facts.md` |
| 2 | 找同赛道样本 | `sample_research.md` |
| 3 | 拆标题、首句、图片结构 | `mimic_brief.md` |
| 4 | 用 skill 做标题和内容诊断 | `title_options.md`、`draft.md` |
| 5 | 做封面和图片方案 | `image_plan.md`、`imagegen_prompt.md` |
| 6 | 去 AI 味和审核 | `pre_publish_check.md` |
| 7 | 整理最终发布稿 | `final_note.md` |

## 每一步的标准

| 环节 | 通过标准 |
|---|---|
| 真实资料 | 数字能追溯，缺的字段明确写缺，不补脑 |
| 对标样本 | 学结构，不抄原句 |
| 标题 | 有地点、预算、人群、风险、反差里的至少两个 |
| 正文 | 有真实判断，有一点生活感，有至少一个风险 |
| 封面 | 用户第一眼知道为什么点进来 |
| 审核 | 没有夸张承诺，没有内部汇报腔 |

## 留痕包结构

每篇内容建一个目录：

```text
YYYYMMDD-[主题]/
  source_facts.md
  sample_research.md
  mimic_brief.md
  title_options.md
  draft.md
  image_plan.md
  imagegen_prompt.md
  pre_publish_check.md
  final_note.md
  generated_images/
```

## 最常见错误

| 错误 | 处理 |
|---|---|
| 一上来就写正文 | 先补真实资料和样本拆解 |
| 标题像广告语 | 改成真实问题或具体条件 |
| 正文太完整太顺 | 删解释，留判断和口语毛边 |
| 封面像 PPT | 换真实图/生图底图，少塞信息 |
| 只吹卖点 | 加一个真实风险或复核项 |
